## DevOps Engineer

- DevOps Engineer zajmuje się łączeniem pracy developerów i zespołów operacyjnych, aby proces wytwarzania oprogramowania był szybki, stabilny i zautomatyzowany.
- Odpowiada za tworzenie i utrzymywanie infrastruktury, automatyzację wdrożeń, monitorowanie systemów oraz dbanie o skalowalność i niezawodność aplikacji.
- Kluczową częścią pracy DevOpsa jest CI/CD, czyli automatyzacja budowania, testowania i wdrażania aplikacji, aby usprawnić procesy i unikać błędów manualnych.
- DevOps dba również o bezpieczeństwo infrastruktury, konfigurację środowisk, zarządzanie zasobami w chmurze oraz szybkie reagowanie w przypadku awarii.
- Rola DevOpsa jest fundamentem nowoczesnych zespołów IT, ponieważ umożliwia szybkie iteracje, stabilne wydania i łatwe skalowanie usług.

## Technologie
- Systemy operacyjne
- Linux (podstawy terminala, zarządzanie procesami, uprawnienia, sieć)
- Bash / Shell scripting
- Chmura (Cloud)
  
### Wybierz jedną platformę i poznaj jej podstawy:
- AWS
- Azure
- Google Cloud Platform
  
### Naucz się:
- tworzyć zasoby (instancje, sieci, bazy)
- zarządzać IAM
- budować proste architektury
- Konteneryzacja
- Docker – budowa obrazów, kontenery, wolumeny, sieci
- Docker Compose – proste środowiska wieloservice’owe
- Orkiestracja kontenerów
- Kubernetes (K8s) — podstawowe obiekty (Pod, Deployment, Service, Ingress)
- Helm (opcjonalnie)
- CI/CD
- Poznaj platformy i narzędzia automatyzacji:
- GitHub Actions
- GitLab CI
- Jenkins
- Infrastructure as Code (IaC)
- Terraform – tworzenie infrastruktury w chmurze jak kod
- Ansible – automatyzacja konfiguracji serwerów
- Monitoring i logging
- Prometheus + Grafana
- ELK / OpenSearch
- Loki

### Przykładowe projekty do portfolio DevOps (samodzielne)
- Dockerizacja prostej aplikacji (np. backend + baza)
- Pipeline CI/CD z automatycznym wdrożeniem na serwer/VM
- Prosta infrastruktura w Terraformie (np. VM + sieć + storage)
- Monitoring aplikacji z Prometheusem i Grafaną
- Deployment aplikacji w Kubernetesie (minikube, k3d)

### Projekty bardziej zaawansowane
- Kompletny pipeline CI/CD z testami, buildem i deploymentem do K8s
- Architektura mikroserwisowa uruchamiana w Kubernetesie
- Auto-skalowanie aplikacji w chmurze (HPA / AWS ASG)
- Canary deployment lub blue-green deployment
- Zbudowanie klastra Kubernetes w chmurze (EKS/GKE/AKS)
- Infrastruktura jako kod w Terraformie dla całego środowiska (VPC, bazy, instancje)

### Co dodać do portfolio?
- kod konfiguracji (Terraform, Ansible),
- pipeline’y CI/CD,
- pliki Dockerfile i manifesty Kubernetes,
- diagramy architektury,
- README z opisem projektów i instrukcją uruchomienia,
- hostowanie projektów w chmurze (AWS/GCP/Azure lub tańsze: Render, Railway),
- screenshoty z panelu monitoringu / logów.
